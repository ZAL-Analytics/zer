API Reference
=============

zer's API reference is generated by ``rustdoc`` and lives alongside this
documentation at ``api/``. Each crate is documented independently; the
table below links directly to the top-level module index for each one.

Key types quick reference
--------------------------

Schema and records
~~~~~~~~~~~~~~~~~~

.. code-block:: rust

   use zer::prelude::*;

   // Build a schema
   let schema = SchemaBuilder::new()
       .field("achternaam",    FieldKind::Name)
       .field("voornamen",     FieldKind::Name)
       .field("geboortedatum", FieldKind::Date)
       .field("postcode",      FieldKind::Id)
       .build()?;

   // Build a record
   let record = Record::new(1)
       .insert("achternaam",    FieldValue::Text("van den Berg".into()))
       .insert("voornamen",     FieldValue::Text("Johannes".into()))
       .insert("geboortedatum", FieldValue::Text("1978-03-15".into()))
       .insert("postcode",      FieldValue::Text("1011AB".into()));

Blocking
~~~~~~~~

.. code-block:: rust

   use zer::prelude::*;  // BlockerFactory, InvertedIndex, Blocker trait

   // Auto-select blocking keys from schema FieldKinds
   let blocker = BlockerFactory::from_schema(&schema);

   // Index all records
   let mut index = InvertedIndex::new();
   for r in &records {
       blocker.index_record(r, &schema, &mut index);
   }

   // Iterate candidate pairs
   for r in &records {
       for candidate_id in blocker.candidates(r, &schema, &index) {
           // compare r with the record whose id is candidate_id ...
       }
   }

Comparison and scoring
~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: rust

   use zer::prelude::*;  // Backend, Comparator, Scorer, Scorer trait

   let backend    = Backend::auto_detect();
   let comparator = Comparator::new(&schema, &backend);
   let scorer     = Scorer::new(&backend);

   // Build a comparison batch from candidate pair indices into a record slice
   let batch = comparator.compare_batch_indexed(&records, &pair_indices, &schema);

   // Estimate EM parameters without any labelled data
   let params = scorer.estimate_params(&batch, None, 200)?;

   // Score a single pair
   let vector = comparator.compare(&rec_a, &rec_b, &schema);
   let scored = scorer.score(&vector, &params);

Pipeline (high-level)
~~~~~~~~~~~~~~~~~~~~~

.. code-block:: rust

   use zer::prelude::*;          // Pipeline, PipelineConfig, ZalEntityStore
   use zer_pipeline::LinkMode;   // not re-exported in prelude

   let pipeline = Pipeline::builder()
       .schema(schema)
       .store(ZalEntityStore::open_in_memory()?)
       .config(PipelineConfig {
           link_mode: LinkMode::Deduplicate,
           ..PipelineConfig::default()
       })
       .build()?;

   let report = pipeline.run_batch(records).await?;
   println!("Entities: {}", report.entities_created);
   println!("Matches:  {}", report.auto_matched);

Generating the API docs locally
---------------------------------

The ``generate_docs.sh`` script builds rustdoc output into
``docs/sphinx/out/api/`` so all crates are browsable from the same
local server:

.. code-block:: bash

   cd /path/to/zer
   bash scripts/generate_docs.sh

   # Serve locally
   python -m http.server 8000 --directory docs/sphinx/out/

Then open ``http://localhost:8000`` for the Sphinx docs and
``http://localhost:8000/api/zer/index.html`` for the rustdoc reference.

Versioning
----------

The API docs are built from the workspace root. All crates share the
same ``1.1`` version defined in their individual ``Cargo.toml`` files.
The ``zer`` facade crate re-exports the subset of the API intended for
downstream consumers, if a type is not re-exported there, treat it as
an implementation detail subject to change without notice.
