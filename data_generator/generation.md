# zer, Data Generation Guide

All synthetic datasets live under `data/` (git-ignored). Run every command from the **repo root**.

Generator scripts live in `data_generator/` and share `data_generator/_common.py`.

---

## One-time setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install faker
```

Subsequent sessions only need `source .venv/bin/activate`.

## Generator scripts

| Script | Primary output | Used by |
|---|---|---|
| `generate_demo_persons.py` | `data/demos/persons/` | `person_deduplication` demo |
| `generate_demo_linkage.py` | `data/demos/linkage/` | `cross_source_linkage` demo |
| `generate_demo_multi_source.py` | `data/demos/multi_source/` | `multi_source_linkage` demo |
| `generate_bench.py` | `data/benchmarks/` | `zer-bench` accuracy / throughput |
| `generate_examples_tests.py` | `data/examples/`, `data/tests/` | crate examples and integration tests |
| `generate_raw.py` | `data/raw/` | raw provider export datasets |

## Generating everything at once

The `scripts/generate_data.sh` wrapper runs all generators in dependency order:

```bash
./scripts/generate_data.sh              # all categories
./scripts/generate_data.sh --demos      # only demo datasets
./scripts/generate_data.sh --benchmarks # only benchmark datasets
./scripts/generate_data.sh --examples --tests  # crate examples + tests
```

## Demo datasets

### persons, single-source deduplication (`data/demos/persons/`)

```bash
python data_generator/generate_demo_persons.py --records 1000 --seed 42
```

Produces `persons.csv` and `ground_truth.csv` with `within_source` duplicate pairs.

### linkage, two-source cross-source linkage (`data/demos/linkage/`)

```bash
python data_generator/generate_demo_linkage.py --persons 600 --seed 7
```

Produces `source_a.csv`, `source_b.csv`, and `ground_truth.csv` with `cross_source` pairs (~40 % overlap).

### multi_source, BRP + KvK, cross-source + within-source (`data/demos/multi_source/`)

```bash
python data_generator/generate_demo_multi_source.py --brp 400 --kvk 300 --seed 11
```

Produces `source_brp.csv`, `source_kvk.csv`, and `ground_truth.csv` with both `cross_source`
and `within_source` match types. Used by the `multi_source_linkage` demo to compare
`LinkOnly` and `LinkAndDedupe` pipeline modes.

Options:
- `--brp N`     , number of BRP (municipal register) records (default 400)
- `--kvk N`     , number of KvK (company director) records (default 300)
- `--overlap F` , fraction of persons shared across sources (default 0.30)
- `--dup-frac F`, fraction of within-source duplicates per source (default 0.08)
- `--seed N`    , random seed for reproducibility (default 11)

BRP record IDs start at 1. KvK record IDs start at 10 000 001 (offset `10_000_000`) to
ensure globally unique IDs when both sources are loaded into the same record store.

## Benchmark datasets (`data/benchmarks/`)

All benchmark datasets are generated by `generate_bench.py` with a `--scenario` flag.
Seeds are fixed so accuracy numbers are reproducible across runs.

```bash
# BRP single-source scenarios
python data_generator/generate_bench.py --scenario brp/dedupe          --seed 41
python data_generator/generate_bench.py --scenario brp/link            --seed 42
python data_generator/generate_bench.py --scenario brp/link_and_dedupe --seed 43

# Cross-schema linking scenarios
python data_generator/generate_bench.py --scenario brp_kvk/link               --seed 44
python data_generator/generate_bench.py --scenario brp_sis/link               --seed 45
python data_generator/generate_bench.py --scenario brp_hks/link               --seed 46
python data_generator/generate_bench.py --scenario brp_kvk_hks/link_and_dedupe --seed 47

# KvK single-source dedupe
python data_generator/generate_bench.py --scenario kvk/dedupe --seed 48

# Micro scenarios for CI smoke tests (~1 K records each)
python data_generator/generate_bench.py --scenario micro/brp/dedupe           --seed 8
python data_generator/generate_bench.py --scenario micro/brp/link             --seed 8
python data_generator/generate_bench.py --scenario micro/brp/link_and_dedupe  --seed 8
python data_generator/generate_bench.py --scenario micro/brp_sis/link         --seed 8
```

Ground truth CSVs use columns: `record_id_a, record_id_b, is_match, match_type`.
